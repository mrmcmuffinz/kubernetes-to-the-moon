# Single-Node Kubernetes Cluster (kubeadm): Overview

A step-by-step guide for bootstrapping a single-node Kubernetes cluster on a QEMU/KVM virtual machine using `kubeadm`. Built for CKA exam preparation as the kubeadm companion to `cka/vm/single-systemd`.

---

## Documents

Follow these in order. The VM creation is reused from `single-systemd/01-qemu-vm-setup.md`.

| # | Document | What It Does | Time |
|---|----------|-------------|------|
| 01 | [Node Prerequisites](01-node-prerequisites.md) | Installs containerd, runc, CNI binaries, crictl, and the `kubeadm`/`kubelet`/`kubectl` toolchain | 15-20 min |
| 02 | [Control Plane Init](02-control-plane-init.md) | Runs `kubeadm init` with a YAML config, sets up `kubectl`, removes the control plane taint | 10-15 min |

| 04 | [Cluster Services](04-cluster-services.md) | Installs Helm, `local-path-provisioner`, `metrics-server` | 5-10 min |

## VM Access

| Access Method | Command |
|--------------|---------|
| SSH into VM | `ssh kube@127.0.0.1 -p 2222` |
| API server from host | `curl -k https://127.0.0.1:6443/healthz` |
| `kubectl` from host | Copy `/etc/kubernetes/admin.conf` from VM, use as kubeconfig |
| VM console log | `tail -f ~/cka-lab/controlplane-1/controlplane-1-console.log` |
| Stop VM | `~/cka-lab/controlplane-1/stop-controlplane-1.sh` |

Default VM credentials: user `kube`, password `kubeadmin`.

## Where Everything Runs

All Kubernetes components run inside the VM. The control plane (etcd, kube-apiserver, kube-controller-manager, kube-scheduler) runs as static pods managed by kubelet via manifests in `/etc/kubernetes/manifests/`. The host machine is used to manage VM lifecycle and optionally to run `kubectl` against the cluster through the port-forwarded 6443.

## Difference from single-systemd

| Concern | `single-systemd` | `single-kubeadm` |
|---|---|---|
| Install method | Manual binaries + systemd units | `kubeadm` |
| Certificates | Hand-generated with cfssl | Generated by `kubeadm init` |
| Control plane processes | systemd services | Static pods managed by kubelet |
| CoreDNS | Manually installed via Helm | Installed by `kubeadm init` |
| CNI | Basic `bridge` plugin | Calico (with `NetworkPolicy` enforcement) |
| Time to working cluster | 2 hours | 10 minutes |

The two guides are complementary, not redundant. `single-systemd` shows what `kubeadm` is doing under the hood. `single-kubeadm` shows the cluster operations workflow you will actually be tested on.

## Scope

Single-node cluster only. The control plane is left untainted so workloads can also schedule on the same node. Multi-node `kubeadm` setup is covered in `cka/vm/two-kubeadm`.
